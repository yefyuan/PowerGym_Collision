# Implementation TODO: Event Class 3 & 4 Gaps — 2026-03-20

**Goal**: Implement condition-triggered events (Class 3) and exogenous disturbance events (Class 4) in Heron's event-driven scheduler. These are the two structural gaps identified in the [event-driven execution guide](event_driven_execution_guide.md), Section 2.3.

**Why these matter**: Without Classes 3 and 4, Heron cannot evaluate the most critical CPS deployment scenario — *how does a trained policy handle an unexpected disturbance arriving mid-cycle?* The full causal chain (disturbance → physics re-solve → condition alarm → agent reacts → delayed effect) is the primary source of compounding degradation at deployment.

**Scope**: No changes to Class 1 (periodic), Class 2 (consequence), or Class 5 (physics) event types. Minor refinements to Class 2 (per-channel message delays) are deferred — acceptable workarounds exist.

---

### 1.2 Condition evaluation hook

**Files**: `heron/scheduling/scheduler.py`

**Why needed**: After physics runs or a disturbance fires, conditions need to be evaluated against the new state to decide whether to trigger agent wakeups.

- [ ] Add `_condition_monitors: List[ConditionMonitor] = []` to `EventScheduler.__init__`

- [ ] Add registration method:
  ```python

      """Register a condition monitor. Conditions are evaluated after
      SIMULATION and ENV_UPDATE events."""
      self._condition_monitors.append(monitor)
  ```

- [ ] Add evaluation method:
  ```python
  def evaluate_conditions(self, proxy_state: Dict) -> List[Event]:
      """Evaluate all registered conditions against current state.
      Schedules CONDITION_TRIGGER events for any that fire.
      Returns list of triggered events."""
  ```
  - Iterate `_condition_monitors`
  - For each: call `monitor.condition_fn(proxy_state)`
  - If `True` and `current_time - monitor._last_triggered >= monitor.cooldown`:

    - Update `monitor._last_triggered = current_time`
    - If `monitor.preempt_next_tick`: call `cancel_events(monitor.agent_id, EventType.AGENT_TICK)`
    - If `monitor.one_shot`: remove from `_condition_monitors`

## Phase 3: Class 4 — Exogenous Disturbance Events

**Depends on**: Phase 1 complete. Phase 2 recommended (for the full causal chain) but not strictly required.

### 3.1 Disturbance data structures

**File**: `heron/scheduling/disturbance.py` (new file)

- [ ] Create:
  ```python
  from dataclasses import dataclass, field
  from typing import Any, Dict, List, Optional
  import numpy as np

  from heron.scheduling.event import Event, EventType
  from heron.agents.constants import SYSTEM_AGENT_ID


  @dataclass
  class Disturbance:
      """A single exogenous disturbance event.

      Attributes:
          timestamp: When the disturbance occurs (simulation time).
          disturbance_type: Domain-specific type string
              (e.g., "line_fault", "load_spike", "generation_drop").
          payload: Domain-specific parameters
              (e.g., {"element": "line_7_8", "action": "disconnect"}).
          requires_physics: If True, an immediate SIMULATION is triggered
              after the disturbance is applied (e.g., topology change
              requires re-solving power flow).
      """
      timestamp: float
      disturbance_type: str
      payload: Dict[str, Any] = field(default_factory=dict)
      requires_physics: bool = True


  class DisturbanceSchedule:
      """A schedule of exogenous disturbances for an episode.

      Can be deterministic (scripted) or stochastic (sampled per episode).
      Enqueued onto the event scheduler at episode reset.
      """

      def __init__(self, disturbances: List[Disturbance]):
          self.disturbances = sorted(disturbances, key=lambda d: d.timestamp)

      @classmethod
      def from_list(cls, specs: List[Dict[str, Any]]) -> "DisturbanceSchedule":
          """Create from list of dicts: [{"t": 12.3, "type": "line_fault", ...}]"""
          return cls([
              Disturbance(
                  timestamp=s["t"],
                  disturbance_type=s["type"],
                  payload={k: v for k, v in s.items() if k not in ("t", "type", "requires_physics")},
                  requires_physics=s.get("requires_physics", True),
              )
              for s in specs
          ])

      @classmethod
      def poisson(
          cls,
          rate: float,
          disturbance_types: List[str],
          t_end: float,
          rng: Optional[np.random.Generator] = None,
      ) -> "DisturbanceSchedule":
          """Generate stochastic disturbance schedule via Poisson process."""
          if rng is None:
              rng = np.random.default_rng()
          disturbances = []
          t = 0.0
          while t < t_end:
              t += rng.exponential(1.0 / rate)
              if t < t_end:
                  dtype = rng.choice(disturbance_types)
                  disturbances.append(Disturbance(
                      timestamp=t,
                      disturbance_type=dtype,
                  ))
          return cls(disturbances)

      def enqueue(self, scheduler) -> None:
          """Place all disturbances as ENV_UPDATE events on the scheduler timeline."""
          for d in self.disturbances:
              scheduler.schedule(Event(
                  timestamp=d.timestamp,
                  event_type=EventType.ENV_UPDATE,
                  agent_id=SYSTEM_AGENT_ID,
                  priority=0,
                  payload={"disturbance": d},
              ))
  ```

- [ ] Add `__init__.py` export in `heron/scheduling/`

### 3.2 ENV_UPDATE handler

**File**: `heron/agents/system_agent.py`

- [ ] Add handler:
  ```python
  @Agent.handler("env_update")
  def env_update_handler(self, event: Event, scheduler: EventScheduler):
      """Handle an exogenous disturbance event.

      1. Extract Disturbance from payload
      2. Apply to environment via domain-specific method
      3. Optionally trigger immediate physics re-solve
      4. Re-evaluate condition monitors
      """
      disturbance = event.payload["disturbance"]

      # Apply disturbance to environment
      self.env.apply_disturbance(disturbance)

      # If disturbance changes state that needs physics (e.g., topology change)
      if disturbance.requires_physics:
          scheduler.schedule_simulation(
              agent_id=self.agent_id,
              delay=0.0,  # immediate
          )

      # Re-evaluate conditions against post-disturbance state
      # (If requires_physics, this happens after SIMULATION instead —
      #  see simulation_handler which already calls evaluate_conditions)
      if not disturbance.requires_physics:
          proxy_state = self.proxy.get_global_states(...)
          scheduler.evaluate_conditions(proxy_state)
  ```

- [ ] Verify the full Class 4 → 5 → 3 → 2 chain fires in order:
  ```

  t=12.3  SIMULATION (immediate re-solve, requires_physics=True)
  t=12.3  ConditionMonitor evaluates → voltage < 0.95 pu detected
  t=12.3  CONDITION_TRIGGER for field_agent_bus_5

  t=12.50 ACTION_EFFECT fires → action applied
  ```
  Total chain latency: 200ms from fault to corrective action.

### 4.3 Evaluation ablation test

- [ ] Compare policy performance across 4 conditions:

  | Condition | Disturbances | Execution mode |
  |---|---|---|
  | A (baseline) | None | Step-based |
  | B | None | Event-driven |

  | D | Fault at t=12.3 | Event-driven (via `ENV_UPDATE`) |
